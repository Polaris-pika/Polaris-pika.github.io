# 多态

方法或对象具有多种形态。是面向对象的第三大特征，多态是建立在封装和继承基础之上的。（抄句定义先）

使用多态机制你可以解决很多代码复用的问题，比如说重写和重载都能体现多态，这是方法的多态

# 向上转型

现在让我们先定义一个父类：animal（我看别人用的这个例子，我也用）

然后我们再定义两个子类：dog 和 cat

并且给他们一个方法和一些变量

然后呢现在去 new 一个我们刚刚定义的类

我们的方法调用的时候是根据我们的运行类型改变的（也就是你 new 的对象），那运行类型当然可以修改

编译类型是我们等号左边的定义（Animal），定义好就不能改了

向上转型：父类的引用指向了子类的对象，可以调用父类中的所有成员（遵守访问权限）

语法：父类类型 引用名 = new 子类类型（）

程序运行的时候会先看看你编译的类型里面有没有这个东西然后再去运行类型里面找他，所以很明显当你的编译类型（Animal）里面没有某个方法的时候（eat（））这时你如果运行这个父类没有的子类方法时，他会给你报错，因为编译器并没有找到对应的方法。

也就是说向上转型不能调用子类的特有成员

但是对于成员变量来说，它并不会根据你的运行类型去找对应的方法，当他被编译的时候就已经确定了他的数值（如果有的话）所以当你调用 animal.number 的时候无论你什么运行类型输出都是 100。（属性是不会被重写的）

# 向下转型

子类类型 引用名 = （子类类型）父类引用；

只能强转父类的引用，不能强转父类的对象。要求父类的引用必须指向的是当前目标类型的对象

也就是说如果你向下转型，那么多半你上面是先向上转型了，这时候你需要子类的某个成员，又转回来了，当向下转型后，可以调用子类类型中所有的成员（运行类型一定要对！）

比如说我们刚刚调用 eat 方法的时候把编译类型转到 Dog，那么当我们调用的时候就可以成功调用了（直接调用方法总不能不成功吧）
